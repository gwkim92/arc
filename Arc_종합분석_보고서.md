## 📋 Executive Summary

Arc는 Circle이 개발한 **USDC 네이티브 가스를 사용하는 허가형 블록체인**으로, 탈중앙화를 희생하고 속도와 규제 준수를 선택한 실용주의적 접근입니다. **2025년 10월 28일 퍼블릭 테스트넷 출시**, 2026년 메인넷 예정이며, B2B 결제 및 금융 인프라 시장을 타겟으로 합니다.

**핵심 특징**:
- 🪙 USDC 네이티브 가스 (가격 변동성 제거)
- ⚡ 1초 미만 결정적 최종성
- 🏢 20개 허가형 검증자 (PoA)
- 🚀 3,000 TPS (20개 검증자 기준)
- 💳 페이마스터 기능 (다중 통화 지원)

**현황**: BlackRock, Visa, Goldman Sachs 등 **100개 이상 기관**이 테스트넷 참여 중

## 1. 가스비 메커니즘 상세 분석

### 1.1 가스비 계산 공식의 본질

#### 일반적인 블록체인 가스비 공식

```
총 가스비 (USD) = g × b_native × P_native
```

- `g` = Gas Used (사용된 가스 단위)
- `b_native` = Base Fee (기본 수수료)
- `P_native` = 네이티브 토큰의 달러 가격

#### 이더리움 예시

```
총 가스비 = 21,000 × 50 Gwei × P_ETH

P_ETH = $2,000 → $2.18
P_ETH = $4,000 → $4.36 (2배 증가!)
```

**문제점**: ETH 가격 변동으로 실제 달러 비용 예측 불가

#### Arc의 혁신

```
CUSD = g × b_USDC × P_USDC
```

**핵심: P_USDC = $1 (고정)**

따라서:
```
CUSD = g × b_USDC × 1
CUSD = g × b_USDC
```

논문 표기 `CUSD = g → bUSDC`는 P_USDC가 $1로 페깅되어 있어 생략한 것입니다.

### 1.2 EIP-1559 메커니즘 차용 및 개선

Arc는 이더리움의 EIP-1559를 기반으로 하되, **두 가지 핵심 개선**을 추가했습니다:

```
총 가스비 = (Base Fee + Priority Fee) × Gas Used
```

- **Base Fee**: 네트워크 혼잡도에 따라 동적 조정, 소각됨
- **Priority Fee**: 검증자에게 지급되는 팁
- **가스 토큰**: ETH가 아닌 USDC

#### Arc의 EIP-1559 개선 사항

**1. Fee Smoothing Mechanism (수수료 평활화)**

Arc는 **EWMA (Exponential Weighted Moving Average, 지수 가중 이동 평균)**을 도입:

```
전통 EIP-1559: 다음 블록 Base Fee = f(직전 블록 사용량)
Arc EIP-1559:  다음 블록 Base Fee = f(여러 블록의 가중 평균)
```

**작동 방식**:
- 과거 블록부터 최근 블록까지 **모든 수수료 데이터 고려**
- 과거 데이터: 지수적으로 감소하는 가중치
- 최근 데이터: 더 큰 가중치 부여

**효과**:
- ✅ Base Fee 변동성 대폭 완화
- ✅ 급격한 수수료 스파이크 방지
- ✅ 기업/기관의 비용 예측성 향상

**2. Base Fee Bounded (수수료 상한선)**

```
Base Fee ≤ 고정된 상한값
```

**효과**:
- ✅ 네트워크 혼잡 시에도 수수료 무한정 상승 방지
- ✅ 최악의 경우에도 예측 가능한 최대 비용
- ✅ 기업 예산 계획 용이

#### 비교: 이더리움 vs Arc

| 항목 | 이더리움 EIP-1559 | Arc EIP-1559 |
|------|------------------|--------------|
| **Base Fee 계산** | 직전 블록만 고려 | 여러 블록 EWMA |
| **변동성** | 높음 (12.5% 변동 가능) | 낮음 (평활화) |
| **상한선** | 없음 | 있음 (Bounded) |
| **예측성** | 중간 | 매우 높음 |

**실제 효과 예시**:

```
이더리움:
블록 N: 50% 사용 → Base Fee 유지
블록 N+1: 100% 사용 → Base Fee +12.5% 급증
블록 N+2: 100% 사용 → Base Fee +12.5% 또 급증

Arc:
블록 N: 50% 사용 → Base Fee 유지
블록 N+1: 100% 사용 → Base Fee 소폭 증가 (EWMA)
블록 N+2: 100% 사용 → Base Fee 완만히 증가 (상한 도달 전까지)
```

### 1.3 비교표

| 구분 | 이더리움 EIP-1559 | Arc |
|-----|-----------------|-----|
| **가스 토큰** | ETH (변동성) | CUSD/USDC (안정) |
| **수수료 메커니즘** | Base + Priority | Base + Priority (동일) |
| **Base Fee 처리** | 소각 | 소각 (동일) |
| **달러 환산** | 변동 | 고정 ($1) |
| **예측 가능성** | ❌ | ✅ |

### 1.4 페이마스터(Paymaster) - 다중 통화 지원의 핵심

Arc의 혁신적 특징 중 하나는 **ERC-4337 기반 페이마스터**를 통한 다중 통화 지원입니다.

#### 페이마스터의 작동 원리

**다른 EVM 체인에서**:
```
사용자 → 네이티브 토큰(ETH) 없음
       ↓
페이마스터 → USDC로 수수료 지불 대행
       ↓
네트워크 → 트랜잭션 실행
```

**Arc에서는 정반대**:
```
사용자 → USDC 없이 다른 스테이블코인 보유
       ↓
페이마스터 → 자동으로 USDC로 변환하여 수수료 지불
       ↓
네트워크 → 트랜잭션 실행
```

#### 지원 가능한 수수료 토큰

Arc는 USDC 외에도 다양한 가치 안정 토큰으로 수수료 지불 가능:

| 토큰 유형 | 예시 | 사용 사례 |
|---------|-----|----------|
| **글로벌 스테이블코인** | EURC (유로) | 유럽 기업 결제 |
| **로컬 스테이블코인** | KRW 스테이블코인 | 국내 시장 특화 |
| **예금 토큰** | Tokenized Deposits | 은행 예금 연동 |
| **CBDC** | Digital Dollar/Euro | 중앙은행 디지털화폐 |

#### 실제 사용 예시

**시나리오 1: 유럽 기업의 FX 결제**
```
독일 기업이 Arc에서:
1. EURC로 거래 수수료 지불
2. 페이마스터가 자동으로 EURC → USDC 환전
3. 네트워크는 USDC로 수수료 수취
4. 기업은 유로 단위로 회계 처리

→ 환전 리스크 제거, 회계 단순화
```

**시나리오 2: 한국 은행의 예금 토큰 활용**
```
한국 은행이 Arc에서:
1. 원화 예금 토큰으로 블록체인 거래
2. 페이마스터가 KRW 토큰 → USDC 변환
3. 실시간 FX 환율 적용
4. 은행은 원화로 정산

→ 크로스보더 결제 자동화
```

#### 기술적 구현: ERC-4337

페이마스터는 Ethereum의 **Account Abstraction** 표준인 ERC-4337을 기반으로 구현됩니다:

**핵심 구성 요소**:
- **UserOperation**: 사용자의 트랜잭션 의도
- **Paymaster Contract**: 수수료 대납 스마트 컨트랙트
- **EntryPoint**: 통합 진입점
- **Bundler**: 거래 묶음 처리

**장점**:
1. ✅ **사용자 경험 개선**: 네이티브 토큰 보유 불필요
2. ✅ **회계 단순화**: 원하는 통화로 수수료 지불
3. ✅ **글로벌 확장성**: 각국 법정화폐 연동 가능
4. ✅ **기관 채택 촉진**: 은행/기업의 기존 회계 시스템과 통합

#### Circle의 전략적 의도

페이마스터 기능은 Arc를 단순한 블록체인이 아닌 **"글로벌 다중 통화 결제 인프라"**로 포지셔닝합니다:

```
Arc의 비전:

모든 법정화폐 스테이블코인 → Arc 페이마스터 → USDC 네트워크
                                              ↓
                                    통합된 글로벌 결제 레일
```

이는 Visa/Mastercard가 여러 통화를 받지만 내부적으로 달러로 정산하는 것과 유사한 모델입니다.

## 4. 블록체인 트릴레마와 권력 집중

### 4.1 블록체인 트릴레마

```
        탈중앙화
          /\
         /  \
        /    \
       /______\
  확장성      보안
```

**3가지 중 2개만 선택 가능**

각 체인의 선택:
- **비트코인**: 탈중앙화 + 보안
- **이더리움**: 탈중앙화 + 보안
- **Arc**: **확장성 + 보안** (탈중앙화 포기)

### 4.2 권력 집중 패턴

| 합의 | 권력 집중 메커니즘 | 현황 |
|-----|----------------|------|
| **PoW** | 해시파워 = 돈 | 4개 채굴풀이 51% |
| **PoS** | 토큰 = 부 | 상위 5개가 32% |
| **PoA** | 허가 = 신뢰 | Circle 100% 통제 |

### 4.3 결론

**PoW, PoS, PoA 모두 트릴레마를 못 벗어나며, 부의 집중으로 인한 권력 집중은 동일합니다.**

- PoW/PoS: 간접적 중앙화 (부의 집중)
- PoA: **직접적 중앙화** (가장 솔직)

## 5. MEV 문제와 Arc의 접근

### 5.1 MEV(Maximal Extractable Value)란?

검증자가 블록 내 거래 **순서를 조작**하여 얻는 이익

### 5.2 주요 MEV 유형

#### 1. 샌드위치 공격 (Sandwich Attack)

```
1. Alice가 10 ETH로 DAI 매수 주문 (Mempool)
2. Bot이 먼저 DAI 매수 (Front-run) → 가격 상승
3. Alice 거래 실행 → 비싼 가격에 구매
4. Bot이 DAI 매도 (Back-run) → 이익 실현
```

**결과**: Alice 손실, Bot 이익

#### 2. 기타 MEV

- **청산 사냥**: DeFi 담보 청산 시 할인가 구매
- **차익거래**: DEX 간 가격 차이 이용
- **타임밴딧**: 블록 재구성 (Reorg)

### 5.3 MEV 규모

- 2023년 이더리움: **$600M+** 추출
- 샌드위치 40%, 청산 30%, 차익거래 20%

### 5.4 Arc의 MEV 완화 전략

> "MEV를 '제거'가 아닌 **'관리 가능'하고 '투명'**하게 만듦"

**전략**:
1. **결정적 순서**: 검증자가 임의로 조작 불가
2. **허가형 검증자**: 평판 리스크, 악의적 행동 시 제재
3. **빠른 최종성**: Reorg 불가능 (타임밴딧 차단)
4. **투명한 감사**: 모든 거래 추적 가능

### 5.5 현실적 평가

**Arc에서도 MEV는 존재합니다**:
- 타이밍 게임 여전히 가능
- 검증자 담합 가능성
- 차익거래는 막을 수도, 막아서도 안 됨

**하지만 이더리움보다 규모는 작을 것**: 
- 예상: $600M/year → $50M/year

## 6. Arc의 차별화 전략 및 생태계 통합

### 6.1 Built-in FX 엔진 - 온체인 외환 결제의 혁신

Arc의 가장 혁신적인 기능 중 하나는 **네트워크에 내장된 기관급 FX 엔진**입니다.

#### FX 엔진의 핵심 기능

**24/7 외환 결제**:
- 전통 금융의 영업시간 제약 없음
- 주말, 공휴일에도 실시간 환전 가능
- 크로스보더 결제 자동화

**하이브리드 구조: 온체인 + 오프체인**

```
┌─────────────────────────────────────┐
│         Arc FX Engine               │
├─────────────────────────────────────┤
│                                     │
│  [온체인]                           │
│  - 스마트 컨트랙트 기반 자동 결제   │
│  - PvP (Payment vs Payment) 정산    │
│  - 투명한 거래 기록                 │
│                                     │
│  [오프체인 RFQ]                     │
│  - Request for Quote 시스템         │
│  - 마켓 메이커 ↔ 기관 환율 견적    │
│  - 가격 비교 & 빠른 체결           │
│                                     │
└─────────────────────────────────────┘
```

#### RFQ (Request for Quote) 시스템

**작동 방식**:

```
1. 독일 은행: "1백만 EURC → USDC 환전 희망"
   ↓
2. Arc FX Engine: 여러 마켓 메이커에게 견적 요청
   - 마켓 메이커 A: 1 EURC = 1.08 USDC
   - 마켓 메이커 B: 1 EURC = 1.082 USDC ← 최선
   - 마켓 메이커 C: 1 EURC = 1.079 USDC
   ↓
3. 최적 환율 자동 선택 (마켓 메이커 B)
   ↓
4. 온체인 스마트 컨트랙트로 즉시 체결
   ↓
5. 1초 미만 완결 확정
```

**전통 외환 거래 vs Arc FX 엔진**:

| 항목 | 전통 외환 | Arc FX 엔진 |
|------|-----------|-------------|
| **운영 시간** | 평일 오전 9시~오후 5시 | 24/7/365 |

#### 주요 사용 사례

**1. 국가 간 결제 (Cross-border Payments)**

## 8. 검증자 확장과 성능 트레이드오프

### 8.1 현재 성능

```
4개 검증자:  10,000 TPS (100ms 블록)
20개 검증자:  3,000 TPS (350ms 블록)
```

**관계**: 검증자 수 ↑ → TPS ↓

### 8.2 BFT 통신 복잡도 O(n²)

| 검증자 수 | 연결 수 | 통신 복잡도 | 예상 TPS |
|---------|--------|-----------|---------|
| 4개 | 6 | 24 메시지 | 10,000 |
| 20개 | 190 | 760 메시지 | 3,000 |
| 50개 | 1,225 | 4,900 메시지 | **1,500** |
| 100개 | 4,950 | 19,800 메시지 | **500** |

### 8.3 왜 확장이 필요한가?

1. **탈중앙화 압력**: 20개는 너무 중앙화
2. **규제 요구**: 최소 50개 권장
3. **보안 강화**: 33% 공격 어렵게
4. **신뢰 분산**: 단일실패점 제거

### 8.4 트레이드오프 딜레마

**옵션 A: 20개 유지**
- ✅ 속도 유지
- ❌ 탈중앙화 비판

**옵션 B: 100개 확장**
- ✅ 탈중앙화
- ❌ TPS 1/6 감소

**옵션 C: 50개 절충 (예상 선택)**
- ✅ 1,500 TPS (충분)
- ✅ 규제 충족
- ✅ 통제력 유지

### 8.5 기술적 해결 방법

1. **Layer 2**: L1 속도 유지, L2로 확장
2. **샤딩**: 검증자 그룹으로 분할
3. **Tendermint 최적화**: 통신 복잡도 감소
4. **하이브리드 합의**: 핵심 + 보조 검증자

### 8.6 현실적 예측

**Arc의 최종 선택: 50개 검증자**

- 1,500 TPS는 B2B에 충분
- 규제 충족 가능
- Circle 통제 가능
- 점진적 확장 (20 → 50 → 유지)

| **확장 시 성능 저하** | ⭐⭐⭐ | BFT O(n²) 한계 |
| **MEV 존재** | ⭐⭐ | 완화만 가능, 제거 불가 |
| **생태계 부족** | ⭐⭐⭐⭐ | 새 체인, dApp 없음 |

### 9.4 SWOT

**Strengths**: Circle 브랜드, USDC 가스, 1초 최종성, 규제 준수  
**Weaknesses**: 중앙화, 생태계 부족, 검증자 확장 한계  
**Opportunities**: B2B 결제, 금융기관 진입, 규제 법안, 크로스보더 송금  
**Threats**: 테더/PayPal 경쟁, 이더리움 L2, Circle 리스크

### 9.5 성공 가능성

- **성공 (40%)**: 10+ 은행 채택, 일일 $10B+, 킬러앱
- **실패 (30%)**: Libra 전철, 채택 느림, 신뢰 하락
- **중간 (30%)**: 틈새 시장, 공존

### 9.6 추천 대상

**추천**:
- ✅ 금융기관
- ✅ B2B 결제 기업
- ✅ 규제 중시 기업

**비추천**:
- ❌ 탈중앙화 순수주의자
- ❌ DeFi 개발자
- ❌ 검열저항 필요 사용자

## 10. 결론

### 10.1 Arc의 의의

Arc는:

1. **스테이블코인 회사의 체인 진입** 트렌드 시작
2. **탈중앙화 vs 실용성** 논쟁에서 실용성 선택
3. **블록체인 트릴레마**에서 탈중앙화 완전 포기
4. **금융 인프라**로서 블록체인의 새로운 가능성 제시
5. **규제 준수**와 블록체인의 양립 가능성 증명

### 10.2 미래 전망 및 로드맵

**Arc 공식 타임라인**:
- **2025년 8월**: 라이트페이퍼 발표 및 Arc 공식 발표 ✅
- **2025년 10월 28일**: 퍼블릭 테스트넷 출시 ✅
  - 100+ 기관 참여 (BlackRock, Visa, Goldman Sachs, HSBC, Mastercard, AWS, Coinbase 등)
- **2026년**: 메인넷 출시 예정
- **2026~2027**: 초기 기관 채택 단계

**장기 전망**:
- **2027~2028**: 스테이블코인 체인 전쟁 본격화 (Tether, PayPal 등 경쟁 체인 출시)
- **2028~2030**: 시장 분화 완성 (기관용 vs 개인용)
- **2030+**: Arc가 Visa처럼 글로벌 결제 인프라가 되는 세상

### 10.3 마지막 질문

**"Arc의 성공은 블록체인의 승리인가, 패배인가?"**

**승리**: 대량 채택, 실제 사용, 메인스트림 진입  
**패배**: 탈중앙화 포기, 이념적 타협, "그냥 DB"

**답**: 둘 다 맞다. **진화이자 변절.**

**확실한 것**: Arc는 블록체인 역사의 **분기점**이며, 우리는 역사적 순간을 목격하고 있습니다.

## 11. 핵심 요약

### 11.1 기술적 혁신

**가스비 공식**:
```
일반: 총 가스비 = g × b_native × P_native
이더리움: 변동 (P_ETH = $1,500~$4,500)
Arc: 고정 (P_USDC = $1)
```

**핵심**: P_native를 $1로 고정하여 가격 변동성 제거

### 11.2 권력 구조

- **PoW**: 4개 채굴풀이 51% (부의 집중)
- **PoS**: 상위 5개가 32% (부의 집중)
- **PoA**: Circle 100% 통제 (허가형 집중)

**결론**: 모두 권력 집중, Arc가 가장 솔직

### 11.3 MEV

- **샌드위치 공격**: Bot이 Front-run/Back-run으로 사용자 착취
- **Arc의 완화**: 결정적 순서, 허가형 검증자, 빠른 최종성
- **현실**: 완화만 가능, 제거 불가

### 11.4 산업 파급력

1. 테더, PayPal도 체인 출시 촉발
2. 기존 L1 (이더리움, 솔라나) 유동성 이탈
3. 금융기관 채택 가속화
4. 탈중앙화 이념 퇴조
5. 시장 분화 (기관용 vs 개인용)

### 11.5 검증자 확장

- **현재**: 20개 → 3,000 TPS
- **미래**: 50개 → 1,500 TPS (예상)
- **한계**: BFT O(n²), 100개 → 500 TPS
- **선택**: 속도 우선, 탈중앙화 포기

### 11.6 최종 평가

**Arc = "블록체인 기술을 쓰는 중앙화 금융 인프라"**

- 기술 혁신: ⭐⭐⭐ (3/5)
- 비즈니스 혁신: ⭐⭐⭐⭐⭐ (5/5)
- 철학적 혁신: ⭐⭐⭐⭐ (4/5)

**성공하면**: 블록체인의 킬러앱, Visa 같은 인프라  
**실패하면**: Libra의 전철, 기업 블록체인 실패 사례

**확률**: 성공 40%, 실패 30%, 중간 30%
